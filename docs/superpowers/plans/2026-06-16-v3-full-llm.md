# v3 — Full-LLM per-segment resolver + tag-multiset safety + live progress

**Goal:** Every flagged segment is checked by the LLM (cost accepted). The LLM ignores genuine
false positives, fixes real errors per the error's content, and only asks the human when it is
NOT 100% confident. A deterministic **tag-multiset guard** makes it impossible to corrupt tags
(never add a foreign tag; enforce per-code count rules). A **live progress UI** shows each segment
as it is processed. Counts are honest and in one unit (issues).

**Stack:** existing `qa_engine`, Claude, pytest, Streamlit. **Repo root:** `memoq-qa-resolver`.

## Decided rules (from user)
- **2016 (order changed):** if target inline-tag multiset == source multiset → **false positive → ignore**. Order may differ for target-language word order. The LLM must NEVER add/remove a tag.
- **2011 missing / 2015 extra:** rule-based to reach **tag-count parity** with source — reinsert missing structural tags (br / `mq:ch` newline / `ph`) near their source position (exact spot doesn't matter, count does); remove extras. Content tags (e.g. `cf color`) keep their source-relative place.
- **Whitespace 3050/3110/3190-3193:** deterministic, always applied (even on mixed segments), fed to the LLM as the candidate target.
- **Other codes:** LLM judges fix vs false-positive and fixes by content.
- **Confidence:** numeric 0-100; auto-apply only at the threshold (default **100**); below → human.
- **Cardinal invariant (all paths):** a proposed target may not introduce an inline tag absent from the source, and must satisfy the per-code count rule; violation → reject → human. Plus existing no-marker-leak + in-memory XML validation.
- **No no-op "fixes":** an item whose written target == current target is not counted as a fix.

### Task 1: Tag inventory + per-code tag rules (`qa_engine/taginv.py`)

Pure functions over **tokenized** segment text (`⟦id:label⟧`) — no XML.

- `tag_multiset(tok_text) -> collections.Counter` keyed by the tag's *kind* (label without volatile ids, e.g. `<cf bold=True size=10>`, `</cf>`, `mq:ch`, `ph`). Use the label part of each token.
- `no_foreign_tags(proposed_tok, source_tok) -> bool`: every tag kind in `proposed` also exists in `source` with count ≤ source count (never invents/duplicates a tag beyond source).
- `count_parity(proposed_tok, source_tok) -> bool`: multisets equal (used for 2011/2015/2016 after fix).
- `tag_verdict_2016(target_tok, source_tok) -> bool`: True when multisets equal → 2016 is a false positive.

**Tests** `tests/test_taginv.py`: equal-count 2016 → FP True; target missing one → False; proposed adds a foreign `<b>` → `no_foreign_tags` False; reordered same tags → `count_parity` True.

Steps: write failing tests → implement → green → commit `feat: tag inventory + per-code tag rules`.

### Task 2: Rewrite per-segment resolver (`qa_engine/resolvers/ai_segment_resolver.py`)

- Schema: `verdict` ("fix"|"false_positive"), `fixed_target` (tokenized), `confidence` (integer 0-100), `rationale`. (Drop the boolean `auto_apply`; confidence drives it.)
- System prompt encodes the decided rules: keep every `⟦id:label⟧` marker; **never add a tag not in the source**; for 2016 with equal tag counts say it's a false positive; for missing/extra structural tags reach count parity placing near source position; fix content per the code's official meaning; give an honest integer confidence (100 only when certain).
- User content includes: each code + `describe_code`, source (tokenized), current target (tokenized), and a **whitespace-normalized candidate** (`collapse_internal_spaces(align_whitespace(...))`).
- After the model returns `fixed_target`: detokenize by id (existing guard). Then apply the **tag-multiset guard** (`taginv`): `no_foreign_tags` must hold; for 2011/2015/2016 `count_parity` must hold. Any violation → `needs_approval=True`, `new_target=None` (cannot safely auto-fix), rationale explains.
- `false_positive` → `action="ignore"` (translation untouched).
- Map confidence: `needs_approval = confidence < threshold` (threshold passed in, default 100). Store numeric confidence.
- Helper `resolve_segment(member, issues, context, ai_client, threshold=100)`.

**Tests** `tests/test_ai_segment_resolver.py` (extend): fake client returning fix@100 → auto; fix@80 → needs_approval; false_positive → ignore; fixed_target adding a foreign tag → guard forces needs_approval + new_target None; 2016 equal-count fake returning false_positive → ignore.

Steps: red → implement → green (update old tests for the new schema/threshold) → commit `feat: full-LLM per-segment resolver with tag-multiset guard + numeric confidence`.
